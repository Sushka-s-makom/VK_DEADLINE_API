# VK Deadline API

Backend-сервис для автоматизации дедлайнов и напоминаний во VK для учебных групп онлайн-школы. Это не чат-бот для личного общения с учениками, а API + worker-инфраструктура, которая хранит группы, дедлайны, правила уведомлений и задачи отправки, а затем рассылает сообщения в общие беседы групп.

Сервис снимает с кураторов рутинную часть работы: им не нужно вручную отслеживать дедлайны, писать однотипные напоминания, подбирать формулировки и контролировать регулярность рассылки. Система сама создаёт задачи на отправку, отправляет напоминания в нужное время и может использовать разные формулировки и изображения, чтобы сообщения не выглядели шаблонно.

## Что делает сервис

- хранит учебные группы, привязанные к VK-беседам;
- импортирует дедлайны из внешних источников;
- создаёт `notification_jobs` по активным правилам уведомлений;
- отправляет напоминания в общий чат группы, а не в личные сообщения ученикам;
- поддерживает нетипичные тексты напоминаний и случайное добавление картинок через VK attachments;
- работает в mock-режиме без реальной отправки, если токен VK не задан.

Основная связка данных: `study_group -> vk_chat -> peer_id`.

## Источники данных

Сервис рассчитан на работу с экосистемой 100Б / 100points:

- учебные группы и дедлайны могут синхронизироваться из курсов 100points.ru;
- для импорта используется аккаунт, у которого есть доступ к нужным курсам на `https://100points.ru` (в том числе корпоративный аккаунт, если через него доступны курсы);
- дедлайны берутся из поля `deadline` у уроков курса;
- каждая беседа привязывается к учебной группе через `peer_id`, после чего рассылка идёт в общий чат этой группы.

В текущем коде есть два реальных источника импорта:

- `100points.ru` через логин/пароль и API `https://api.100points.ru/api`;
- `Google Sheets` как альтернативный источник для MVP и ручных сценариев.

Отдельного клиента для личных сообщений ученикам или отдельной сущности "ученик" в сервисе нет.

## Как устроена рассылка

Рассылка идёт в групповые чаты VK. Это сделано намеренно: с точки зрения процессов кураторов общий чат группы целесообразнее, чем поддержка отдельных диалогов с каждым учеником. При этом архитектура выдерживает отправку и в другие чаты, если такие привязки будут заведены.

Текст сообщений собирается динамически из набора шаблонов:

- приветствие;
- подводка;
- детали дедлайна;
- мотивирующая фраза;
- завершающая строка.

За счёт этого напоминания получаются разнообразными, а не одинаковыми "сухими" сообщениями. Также сервис умеет иногда прикладывать картинку через сохранённый VK attachment.

## Стек

### Используется в текущем backend-сервисе

- Python 3.12
- FastAPI
- Uvicorn
- Pydantic 2 / pydantic-settings
- SQLAlchemy 2
- Alembic
- SQLite для локального запуска, PostgreSQL поддержан через `DATABASE_URL`
- Redis как broker/backend для очередей
- Celery worker + Celery beat
- `requests`
- `urllib.request` для части импортов
- `python-dotenv`
- VK API `messages.send`

### Есть в репозитории, но не является основой текущего сервиса

- каталог `vk_deadline_bot/` выглядит как более ранний отдельный бот/MVP и не используется как основной runtime для `app/`;
- ручные служебные скрипты в `scripts/` используются для импорта, диагностики, сидирования и локального запуска отправки.

### Чего в текущем репозитории нет

- нет модели учеников и штатной рассылки в личные сообщения: сейчас это сознательно не реализовано, чтобы не захламлять личные диалоги учеников однотипными напоминаниями, хотя технически такой сценарий можно добавить;
- нет отдельного фронтенда;
- нет выделенного Telegram-бота или VK-бота для диалогового общения;
- нет `docker-compose.yml` в корне репозитория, поэтому `Docker Compose` нельзя считать подтверждённой частью текущей поставки проекта.

## Импорт дедлайнов из 100points.ru

Настройте `.env`:

```env
POINTS100_EMAIL=почта@100points.mail
POINTS100_PASSWORD=пароль
POINTS100_COURSE_IDS=1872,2536
POINTS100_COURSE_CHATS=1872:2000000001,2536:2000000002
POINTS100_DEFAULT_PEER_ID=2000000002
```

Запустить импорт только будущих дедлайнов:

```bash
.venv/bin/python -m scripts.import_100points_deadlines
```

Импортировать всё, включая прошлые дедлайны:

```bash
.venv/bin/python -m scripts.import_100points_deadlines --include-past
```

Переопределить курсы без правки `.env`:

```bash
.venv/bin/python -m scripts.import_100points_deadlines --course-ids 2536,2753
```

Что делает импорт:

- логинится в `100points.ru`;
- получает список уроков курса;
- создаёт или обновляет `StudyGroup` по курсу;
- привязывает группу к нужному `peer_id`;
- создаёт или обновляет дедлайны и задачи на отправку.

Если у аккаунта нет доступа к курсу, такой курс будет пропущен.

## Импорт дедлайнов из Google Sheets

Для MVP в репозитории остаётся альтернативный импорт из Google Sheets. Скрипт читает темы из строки 1 начиная с `H1`, дедлайны из строки 4 начиная с `H4`, создаёт группу `Группа eb-046`, чат `General` с `peer_id=2000000002`, правила уведомлений и `notification_jobs`.

```bash
.venv/bin/python -m scripts.import_google_sheet_deadlines
```

Импортировать и прошлые даты:

```bash
.venv/bin/python -m scripts.import_google_sheet_deadlines --include-past
```

## Настройка "живых" и разнообразных напоминаний

Переключить MVP на живые сообщения и дневное окно отправки `13:00-15:00`:

```bash
.venv/bin/python -m scripts.configure_fun_deadline_messages
```

После этого:

- активным остаётся правило `В день дедлайна, окно 13:00-15:00`;
- текст каждого сообщения начинается с `#дедлайн`;
- сообщение собирается из случайных фраз по категориям;
- время внутри окна выбирается стабильно для каждой связки `deadline + vk_chat + rule`.

Добавить картинку через готовый VK attachment:

```bash
.venv/bin/python -m scripts.add_vk_media_asset photo-123456_789012 --title "Deadline image"
```

При отправке сервис может случайно прикрепить такую картинку к сообщению.

## Mock-режим VK

Если переменная окружения, указанная в `vk_communities.token_env_name`, не задана или пустая, `VKSender` не падает и не вызывает VK API. Вместо этого отправка логируется как mock.

## Реальная отправка во VK

1. Создайте токен сообщества VK с правом отправки сообщений.
2. Укажите токен в `.env`, например `VK_GROUP_TOKEN=...`.
3. Убедитесь, что в базе у сообщества указано `token_env_name = "VK_GROUP_TOKEN"`.
4. Убедитесь, что сообщество добавлено в нужные беседы и имеет право писать сообщения.

Найти `peer_id` беседы можно так:

```bash
.venv/bin/python scripts/find_vk_peer.py
```

## Основные endpoints

- `GET /health`
- `POST /vk-communities`, `GET /vk-communities`, `GET /vk-communities/{id}`
- `POST /study-groups`, `GET /study-groups`, `GET /study-groups/{id}`
- `POST /vk-chats`, `GET /vk-chats`, `GET /vk-chats/{id}`
- `POST /deadlines`, `GET /deadlines`, `GET /deadlines/{id}`, `PATCH /deadlines/{id}`
- `POST /notification-rules`, `GET /notification-rules`, `PATCH /notification-rules/{id}`
- `GET /notification-jobs`, `GET /notification-jobs/{id}`

После создания дедлайна сервис автоматически создаёт `notification_jobs` для активных правил и активных VK-бесед учебной группы. Дубли запрещены уникальным индексом `deadline_id + vk_chat_id + rule_id`.
